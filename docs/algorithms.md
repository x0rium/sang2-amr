# Алгоритмы SANG2 → AMR: детальный маппинг

## Таблица адаптации

Каждый алгоритм auto-reverser переносится в геномный домен
с минимальной адаптацией (смена алфавита и единицы потока).

---

## A1. Shannon Entropy (H)

### В auto-reverser
`H = −Σ P(x) log₂ P(x)` по байтовым позициям в пакетах.
Классифицирует: padding / structured / encrypted.

### В AMR
Скользящее окно по контигу (window = 100–500 nt, шаг = 1).

**Интерпретация:**
- **Низкая H (< 1.0 bit)**: гомополимерные треки, простые повторы, padding-аналоги
- **Средняя H (1.2–1.8 bit)**: кодирующие регионы (codon bias снижает энтропию)
- **Высокая H (≈ 2.0 bit)**: интергенные регионы, интроны (близко к максимуму для 4-буквенного алфавита)
- **Аномально низкая H в контексте высокой у соседей**: insertion sequence / мобильный элемент с другим GC-составом

**Для AMR:** Резкий перепад H на границе региона = кандидат в горизонтально перенесённый фрагмент (Genomic Island). Резистентные гены часто на плазмидах/транспозонах с GC-составом, отличным от хозяина.

**Формула адаптации:**
- `H_max = log₂(4) = 2.0` (для нуклеотидов)
- `H_max = log₂(20) ≈ 4.32` (для аминокислот)

## A2. SDR (Structural Detection Rule)

### В auto-reverser
Binary search для θ, где символы с `freq/total > θ` несут ~50% массы.
SDR ratio ≈ 5–7% словаря = «структурные» байты.

### В AMR
Применяется на уровне **k-меров** (k = 3..8), не отдельных нуклеотидов
(алфавит из 4 букв слишком мал для SDR).

**Интерпретация:**
- Структурные k-меры = консервативные мотивы (Shine-Dalgarno, стартовые кодоны, промоторные элементы -10/-35)
- Контентные k-меры = вариабельные последовательности

**Для AMR:** SDR на k-мерах из зоны предполагаемого мобильного элемента:
если SDR-профиль резко отличается от хозяйского генома → подтверждение чужеродного происхождения.

## A3. Otsu Threshold

### В auto-reverser
Бинарное разделение непрерывных распределений без magic numbers.

### В AMR
Применяется повсеместно:

1. **GC-content bucketing**: Otsu на GC% по окнам → автоматическое разделение на изохоры / чужеродные вставки
2. **Coverage bucketing**: Otsu на глубине покрытия → плазмидные контиги (обычно higher copy number) vs хромосомные
3. **Length bucketing**: Otsu на длинах контигов → фильтрация мусорных коротких контигов
4. **MIC bucketing** (Mode C): Otsu на значениях MIC → resistant / susceptible без CLSI breakpoints

**Ключевой принцип:** CLSI/EUCAST breakpoints для MIC — это magic numbers, установленные комитетом. Otsu выводит порог из распределения данных. Это не замена клинических стандартов, но независимая проверка.

## A4. Bidirectional Entropy (ΔH)

### В auto-reverser
`ΔH[i] = H(next | past_k) − H(prev | future_k)`
- ΔH < 0 → конвергентное → структурное (headers)
- ΔH > 0 → дивергентное → контент (payload)

### В AMR
Скользящее окно по контигу, context_k = 8..32 нуклеотидов.

**Интерпретация:**
- **ΔH < 0 (конвергентное)**: начало гена, промотор, RBS (Ribosome Binding Site) — следующий нуклеотид предсказуем из предыдущих (codon bias, conserved start)
- **ΔH > 0 (дивергентное)**: конец гена, терминатор, интергенный регион — предыдущий контекст не предсказывает следующее
- **Резкий переход ΔH<0 → ΔH>0**: граница гена

**Для AMR:** Кассеты резистентности (gene cassettes в интегронах) имеют характерный паттерн: attC (convergent, ΔH<0) → gene (variable) → attC (convergent). ΔH-профиль найдёт интегрон без знания последовательности attC.

## A5. MI Half-life (Ключевой алгоритм)

### В auto-reverser
`MI(lag) = 2·H_global − H_joint(X_i, X_{i+lag})`
Разделение 86.6σ между типами данных.
English text ≈ 8, Python ≈ 21, PCAP ≈ 55, WAV ≈ 2, PNG ≈ 4.

### В AMR
**Предсказания (требуют экспериментальной проверки):**

| Тип последовательности       | Ожидаемый MI half-life | Почему                              |
|------------------------------|------------------------|-------------------------------------|
| Кодирующая ДНК               | ≈ 3 nt                | Codon structure (триплеты)          |
| Интрон / интергенный         | > 10 nt               | Нет кодонового ограничения          |
| rRNA гены                    | ≈ 3–5 nt              | Высокая структурная консервативность |
| IS-элемент (insertion seq.)  | ≈ 3 nt, но другой профиль | Кодирующий, но другой codon usage  |
| CRISPR spacer                | >> 10 nt               | Захваченная чужеродная ДНК, вариабельная |
| Тандемные повторы             | = длина повтора        | Периодичность по определению        |

**Для AMR:**
- MI half-life ≈ 3 в некодирующем контексте → вероятно, незамеченный ORF (кандидат в новый ген резистентности)
- MI half-life хозяйского генома vs вставки — разный codon usage → разный decay profile

**Результат exploration/001 (2026-04-12): ГИПОТЕЗА НЕ ПОДТВЕРДИЛАСЬ.**

MI half-life для CDS = 37.9 (ожидалось 2.5–4.0). С 4-буквенным алфавитом MI(lag=1)
составляет лишь 1.2% от H_max — недостаточно для экспоненциального затухания.
Кодонный сигнал (период 3) детектируется через **автокорреляцию** (97.2% CDS, сигнал A6),
а не через MI decay. MI(3)>MI(4) в 66% CDS vs 53% intergenic — слабый, но значимый.
FFT MI-кривой показывает доминирование period-6 (кодон-пара), не period-3.

**Замена: Autocorrelation Periodicity Index** — наличие и сила пиков автокорреляции
на кратных целевого периода (3 для кодонов, N для повторов длины N).
Подробности: exploration/003_tools.md, gotchas G15.

## A6. Autocorrelation + Peak Detection

### В auto-reverser
Пики при `ac[i] > ac[i±1] and ac[i] > 0.03`.
Код: lag 3,6,9 (индентация). PCAP: lag 3,15,22 (поля).

### В AMR
**Интерпретация пиков:**
- **Lag 3**: кодонная структура (базовый сигнал кодирующих регионов)
- **Lag 6, 9**: подтверждение кодирующего региона (гармоники)
- **Lag N (N > 3, не кратно 3)**: тандемные повторы длины N
- **Отсутствие lag-3 пика**: некодирующий регион

**Для AMR:**
- Тандемные повторы (VNTR) используются в типировании бактерий (MLVA). Автокорреляция найдёт их без предзнания длины повтора.
- Insertion sequences имеют характерные inverted repeats на концах → пик автокорреляции на lag = длина IS.
- CRISPR-массивы: повтор (≈30 nt) + спейсер (≈30 nt) → пик на lag ≈ 60.

## A7. H/H_max Regime Classifier

### В auto-reverser
4 режима: SPARSE, RAW_SIGNAL, SYMBOLIC, HIGH_ENTROPY.

### В AMR
Адаптация границ для 4-буквенного алфавита (H_max = 2.0):

| H_ratio        | Режим          | Геномная интерпретация                              |
|----------------|----------------|------------------------------------------------------|
| 0.00–0.25      | SPARSE         | Гомополимерные треки, poly-A/T хвосты                |
| 0.25–0.60      | BIASED         | Экстремальный GC-bias (GC-богатые/AT-богатые регионы)|
| 0.60–0.85      | CODING         | Кодирующие регионы с codon bias                      |
| 0.85–1.00      | BALANCED       | Интергенные, высоковариабельные                      |

**Для AMR:**
- Мобильные элементы часто имеют **другой режим**, чем хозяйский геном
- Pathogenicity islands: островок CODING в окружении BALANCED = подозрительный
- Некоторые гены резистентности (например, к ванкомицину, vanA) имеют аномальный GC → переход между режимами

## A8. Hurst Exponent

### В auto-reverser
R/S rescaled-range. H > 0.5 → persistent (structured).

### В AMR
**Baseline:** ДНК имеет H ≈ 0.58–0.67 (Peng et al., 1992 — long-range correlations в интронах; Li & Kaneko, 1992 — кодирующие регионы).

**Применение:**
- Скользящий Hurst по контигу. Резкое изменение H → граница genomic island
- Горизонтально перенесённая ДНК: H источника ≠ H хозяина
- Плазмидная ДНК vs хромосомная: разный Hurst из-за разных эволюционных давлений

**Ограничение:** Hurst требует длинных последовательностей (>= 256 nt для стабильной оценки). На коротких ридах (150 nt Illumina) не работает — только на контигах/скаффолдах.

## B1. Pair Merge (Hierarchical Compression)

### В auto-reverser
`score(a, b) = PPMI(a,b) × log₂(1 + count(a,b))`
4.6M bytes → 785 tokens, 8 levels. Lossless.

### В AMR
**Иерархия слияния для нуклеотидов:**

```
Level 0: A, C, G, T                    (4 символа)
Level 1: Динуклеотиды (CG, AT, ...)   (высокий PPMI = codon-dependent)
Level 2: Кодоны (ATG, TAA, ...)        (функциональные единицы)
Level 3: Мотивы (TATAAT = -10 box)     (регуляторные элементы)
Level 4: Домены (~100-300 aa)           (структурные/функциональные единицы белка)
Level 5: Гены (~300-3000 nt)           (полные кодирующие последовательности)
Level 6: Опероны / кассеты             (ко-регулируемые группы)
Level 7: Pathogenicity islands          (крупные мобильные блоки)
```

**Для AMR:**
- SDR на score-distribution автоматически определяет «где кончается шум и начинается структура» на каждом уровне
- Кассеты резистентности (integron gene cassettes) должны появиться как coherent единицы на Level 6
- **Lossless reversibility**: можно пройти от «pathogenicity island» вниз до конкретных нуклеотидов

**Результат exploration/003 (2026-04-12): PPMI scoring не работает на 4-буквенном алфавите.**

PPMI × log2(1+count) создаёт runaway chains: merged symbols редки → PPMI зашкаливает →
каскадное расширение одной цепочки (GC → GGC → GGCG → ...). Только 4/16 dinucleotides.

**Замена: Standard BPE (frequency-only).** 300 merges, compression 3.62x, lossless.
Автоматически строит иерархию: nt → dinucleotides (11/16) → codons (23/64, включая ATG, TAA, TAG) → 4-mers → 5-mers. PPMI используется POST-HOC для анализа, не для scoring.

Подробности: exploration/003_tools.md, gotcha G18.

## R. Resonance (6 сигналов, геометрическое среднее)

### В auto-reverser
`R(u) = ∏ sᵢ^(1/k)` для пар (b1, b2, k).
Минимальная единица = relational (≥2 DOF).

### В AMR
Минимальная единица: **динуклеотид** или **пара кодонов** (не single nucleotide).

**Адаптация 6 сигналов:**

| Сигнал | auto-reverser          | AMR                                                |
|--------|------------------------|----------------------------------------------------|
| A      | Частота пары           | Частота динуклеотида/к-мера в геноме                |
| D      | Направленность (→ vs ←)| Strand bias: CG→ vs GC→                            |
| MI     | PPMI пары              | PPMI динуклеотида (CpG suppression = низкий MI)    |
| P      | Позиционная стабильность| Привязка к позиции внутри кодона (1st/2nd/3rd)     |
| X      | Контекстная стабильность| Предсказуемость окружающих нуклеотидов              |
| G      | Gap consistency        | Что между парами на расстоянии k (= motif spacing) |

**Для AMR:**
- **Высокий R** на паре кодонов = консервативный функциональный элемент (active site, binding pocket)
- **Eigendistances** (пиковые k) = характерные расстояния внутри структуры:
  - k=3: внутрикодонное расстояние
  - k=10-12: альфа-спиральный шаг (3.6 aa × 3 nt/aa ≈ 11 nt)
  - k=~30: минимальный домен
- **Resonance spike** в нетипичной позиции генома = кандидат в функциональный элемент

## Cross-Reference → HGT Detection

### В auto-reverser
3 механизма: constant echo, family tag echo, session value correlation.

### В AMR
Прямой маппинг на горизонтальный перенос генов (HGT):

| auto-reverser           | AMR                                                     |
|-------------------------|---------------------------------------------------------|
| Constant echo           | Идентичная последовательность в разных изолятах/видах   |
| Family tag echo         | Общий домен/мотив (β-лактамазный fold) в разных генах   |
| Session value correlation| Ко-резистентность: MIC к препарату A ↔ MIC к препарату B|

**Это ключевой модуль для AMR:** мобильные генетические элементы (плазмиды, транспозоны, интегроны) обнаруживаются именно как cross-references между разными «сессиями» (изолятами).

## Clustering → Reference-free Taxonomic Grouping

### В auto-reverser
Length bucketing (Otsu) + discriminator columns + signature.

### В AMR
1. **Length bucketing**: контиги по длине → хромосомные vs плазмидные
2. **Discriminator positions**: нуклеотидные позиции с малым vocab (1 < |unique| ≤ 4) → таксономически информативные SNP
3. **Signature**: (bucket, discriminator_values) → кластер = OTU-аналог

**Для AMR:**
- Mode B (metagenome): кластеризация контигов по организму-источнику без референсной базы
- Mode C (cohort): кластеризация изолятов по генотипу резистентности

## Что НЕ переносится напрямую

| Алгоритм auto-reverser | Причина                                                       |
|-------------------------|---------------------------------------------------------------|
| Stream parsing (length-prefix) | Геномные форматы (FASTA/FASTQ) фиксированы, не нужно угадывать |
| Checksum detection (CRC/XOR)   | Нет прямого аналога в ДНК (но conserved flanks — частичный) |
| FSM assembly                    | Гены не образуют конечный автомат; нужна другая модель       |
| Token-based parsing (FIX-like)  | Нет Key=Value структуры в нуклеотидных последовательностях  |
