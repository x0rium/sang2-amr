# Подводные камни (Gotchas)

## Алфавит и энтропия

### G1. H_max для ДНК = 2.0, не 8.0
Байтовый алфавит: H_max = log₂(256) = 8.0 бит.
Нуклеотидный: H_max = log₂(4) = 2.0 бит.
Аминокислотный: H_max = log₂(20) ≈ 4.32 бит.

Все формулы, использующие H_max, должны быть параметризованы по алфавиту.
SDR thresholds, regime boundaries, H_ratio — всё пересчитывается.

### G2. GC-bias ≠ низкая энтропия
Геном с GC = 70% имеет неравномерное распределение нуклеотидов,
но это норма для организма (Streptomyces, Deinococcus). Не путать с padding.
Нужно нормализовать H по ожидаемому GC-составу хозяина.

### G3. Complementary strand
ДНК двуцепочечная. Последовательность одной цепи детерминирует другую.
При анализе пар нуклеотидов: CG на одной цепи = GC на другой.
Некоторые сигналы (D — направленность) могут давать артефакты,
если не учитывать strand symmetry.

## Биологические особенности

### G4. Codon degeneracy confounds MI
3-я позиция кодона (wobble) часто вариабельна без изменения аминокислоты.
MI на нуклеотидном уровне будет ниже, чем на кодонном.
Решение: вычислять MI на обоих уровнях (nt и codon), сравнивать.

### G5. Repeats inflate autocorrelation
Бактериальные геномы содержат rRNA-опероны (5–15 копий по ~5 kb),
IS-элементы (десятки копий). Они создают сильные пики автокорреляции,
не связанные с AMR. Нужен blacklist или отдельная обработка repetitive regions.

### G6. Горизонтальный перенос ≠ AMR
Не все мобильные элементы несут гены резистентности.
Bacteriophages, metabolic islands, fitness islands — всё это будет
определяться теми же сигналами (Hurst anomaly, regime shift).
Нужен дополнительный фильтр для AMR-специфичных паттернов
(proximity к known AMR, co-occurrence с integrase/transposase).

### G7. Paralogous genes confound cross-reference
Гомологичные гены внутри одного генома (паралоги) выглядят как
cross-reference, но это не горизонтальный перенос.
Нужно различать: same genome (paralog) vs different genomes (HGT).

## Технические ограничения

### G8. Short reads (Illumina 150 bp) vs long reads
- MI half-life: нужен контекст >>  half-life. Для coding DNA (hl ≈ 3), 150 bp достаточно.
  Для higher-order structure — нет. Работать с контигами, не с ридами.
- Hurst: нужно ≥ 256 точек. На ридах не работает.
- Pair merge: на ридах даст артефакты от фрагментации. Только на контигах.

### G9. Metagenome assembly fragmentation
Метагеномная сборка часто фрагментирована: короткие контиги (< 1 kb),
химерные контиги (два организма склеены). Это создаёт:
- Ложные regime shifts на стыках
- Ложные Hurst anomalies
- Clustering по организмам может не работать для < 2 kb контигов

Митигация: фильтр по длине контига (min 2 kb для полного анализа),
reduced signal set для коротких (только H, SDR, autocorrelation).

### G10. Circular genomes
Бактериальные хромосомы и плазмиды — кольцевые.
Линеаризация создаёт артефактную границу.
При анализе скользящим окном — wrap-around на концах.

### G11. Strand-specific signals
Гены могут быть на любой цепи (+/-). Анализ должен учитывать
reverse complement. Для ΔH и MI — проверять оба направления.

## Вычислительные

### G12. Размер данных
Один бактериальный геном: 1–10 Mb (тривиально).
Метагеномная проба: 1–100 Gb ридов, 10–500 Mb контигов.
Cohort из 10,000 изолятов: 50–100 Gb контигов.
Vectorized numpy справится, но нужен батчинг для cohort.

### G13. k-mer space explosion
Для SDR на k-мерах: 4^k возможных к-меров.
k=8 → 65,536 (ок). k=12 → 16M (тяжело). k=16 → 4B (невозможно в памяти).
Решение: sparse representation (Counter / scipy.sparse), не dense array.

## Валидационные

### G14. Нет ground truth для «novel» AMR
По определению, новые механизмы резистентности не аннотированы.
Валидация novel detection — самая сложная часть.
Подходы: leave-one-out (G14a), retrospective (G14b), synthetic insertion (G14c).
Детали в data-sources.md.

### G15. MI half-life не переносится на 4-буквенный алфавит

**Обнаружено в exploration/001 (2026-04-12).**

MI half-life — ключевой сигнал auto-reverser (86.6σ разделение типов данных) —
**не работает** на нуклеотидном алфавите. Причина:

- С 256 символами MI(lag=1) типично 20–60% от H_max → чёткое экспоненциальное затухание
- С 4 символами MI(lag=1) = 0.024 bits = 1.2% от H_max → нет затухания, осцилляция на уровне шума
- Measured MI half-life для CDS: **37.9** (ожидалось 2.5–4.0)

Кодонная структура (период 3) **видна** через автокорреляцию (97.2% CDS)
и слабо через MI(3)>MI(4) (66% CDS vs 53% intergenic).

**Решение:** Заменить MI half-life на "autocorrelation periodicity index":
- Пики автокорреляции на lag=3k → кодирующий регион
- FFT MI-кривой для обнаружения периодичности
- На кодонном уровне (алфавит 64) MI может работать лучше, но тоже не затухает —
  коррелированность кодонов остаётся высокой на всех лагах (белковая композиция)

### G16. H/H_max regime thresholds не масштабируются на маленький алфавит

**Обнаружено в exploration/001 (2026-04-12).**

Для 4 символов с GC≈50%: H/H_max ≈ 0.99 для ВСЕХ регионов (CDS и интергенных).
Пороги 0.25/0.60/0.85 из байтового домена бессмысленны.

**Решение:** Использовать 3-mer entropy (эффективный алфавит 64):
- CDS: 3-mer H_ratio = 0.966 (все 64 кодона задействованы, codon bias умеренный)
- Intergenic: 3-mer H_ratio = 0.914 (AT-bias сужает используемый набор)
- Otsu порог: 0.917

Направление **инвертировано** по сравнению с байтовым доменом:
CDS = высокая 3-mer энтропия, IG = низкая (в байтовом: structured = низкая, content = высокая).
Причина: кодирующие регионы используют широкий набор кодонов, а некодирующие —
ограниченный из-за compositional bias.

### G17. PPMI pair merge создаёт runaway chains на маленьком алфавите

**Обнаружено в exploration/003 (2026-04-12).**

score(a,b) = PPMI(a,b) × log₂(1+count(a,b)) — формула из SANG2 — ломается на |Σ|=4.
После первого merge (GC → новый символ), P(GC) мала → PPMI любой пары с GC зашкаливает →
алгоритм расширяет одну цепочку вместо создания разнообразных единиц.

На 256-символьном алфавите P(x) ≈ 1/256, и PPMI работает в правильном динамическом диапазоне.
На 4-символьном P(x) ≈ 1/4, и любой merged symbol сразу становится «редким» относительно базовых.

**Решение:** Использовать frequency-based BPE (standard). Компрессия 3.62x, 23/64 кодона найдены.
PPMI полезен для POST-HOC анализа (какие merges «структурные»), но не для scoring.

**Паттерн:** Это третий случай (после MI half-life G15 и H/H_max G16), где формула SANG2
ломается из-за маленького алфавита. Общее правило: любая формула с P(x) в знаменателе
(PMI, PPMI, surprise) будет давать экстремальные значения при |Σ| ≤ 20.

### G18. Survivorship bias в CARD
CARD содержит преимущественно клинически значимые гены из клинических изолятов.
Экологическая резистентность (soil, water) представлена слабо.
SANG2-AMR может находить «новые» гены, которые на самом деле известны
в экологических базах (но не в CARD).
Проверять против: MEGARes, ARDB, ResFams в дополнение к CARD.
